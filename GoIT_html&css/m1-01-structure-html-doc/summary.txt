Module 01-02 Structure of HTML document
	1. Смысловые обертки <header>, <nav>, <main>, <section>, <footer> - для группировки контента по смыслу.
	2. Тег <article> - уникальный контент, которому можно дать заголовок и вынести со страницы, и если перенесем на другую страницу, то он там будет иметь смысл. Абсолютно независимый блок контента. Например, главный баннер с рекламой. Если можно дать имя разделу и вынести его на другой сайт - это <article>. В практике встречается редко.
	3. Тег <section> - это секция какого-то однотипного контента, связанного. У секции должен быть заголовок. Если у секции нет заголовка, то это не правильно, лучше поставить заголовок и скрыть его. Это нужно для скринРидеров. Если можно дать имя разделу, но вынести на другой сайт нельзя, не будет иметь смысла, зависит от текущий старницы, то это <section>.
	4. Тег <div> - если нельзя дать вменяемое имя контенту (не левая колонка), а смысловое. Что такое <div>? Это блок, а что такое блок? Ничто, пустое место для скрин ридера, отсутствие семантики.
	5. Есть блочные элементы, а есть строчные элементы.
	6. В <ul> и <ol> можно вложить только <li> 
	7. В li можно вложить что-угодно - <p>, <a>, <div>
	8. Семантическая разметка - это разметка сайта, которая имеет смысл. Несемантическая (сайты-интерфейсы) - состоит из <div> везде.
	9. Правильная разметка - сильно увеличивает конверсию для поисковых роботов и скрин ридеров, другими словами увеличивает деньги, которые приносит сайт.
	10. Крупные смысловые разделы: <article>, <section>, <nav>, <div>
	11. Потоковые теги: <header>, <footer>, <main>, <ul>, <ol>, <li>, <p>
	12. Фразовые теги: <img>, <a>, <button>, <br>, <span>
	13. <img> - это картинка. Картинки бывают фоновые и оформительные. Контентные - которые стоят непосредственно внутри html и несут смысл, важна для пользователя. Атрибут alt - это альтернативный текст, который увидит пользователь, если картинка не загрузиться. Оформительные - это фоновые изображения, которые сделаны внутри css.
<id> - это специальный атрибут, который задает уникальный идентификатор ресурса на странице. <id> больше относиться не к css, а к html. <id> нужно использовать для навигации по текущей странице, если такая необходимость есть - <a href="#services" class="main-menu-link">services</a>.
